LeetCode 25：K 个一组翻转链表

链表分组、原地翻转、边界保护。先找到完整的一组，把这一组临时断开，复用熟悉的反转链表函数，再把前后链路接回去。

题目概述

给定链表头节点 head 和整数 k，每 k 个节点翻转一次，最后不足 k 个节点保持原顺序。题目要求不能只交换节点值，必须真实改变节点连接。

链表 双指针 原地修改 分组处理 迭代

示例拆解

示例 1

head = [1,2,3,4,5], k = 2

每两个节点一组：[1,2] 变成 [2,1]，[3,4] 变成 [4,3]，剩余 [5] 不足一组，输出 [2,1,4,3,5]。

示例 2

head = [1,2,3,4,5], k = 3

前三个节点翻转成 [3,2,1]，尾部 [4,5] 不足 k 个，保持不动，输出 [3,2,1,4,5]。

约束：链表节点数 n 满足 1 <= n <= 5000，节点值 0 <= Node.val <= 1000，并且 1 <= k <= n。

考察点

这道题考察链表原地改指针、分组边界识别、哑节点简化头节点变化，以及对“尾部不足 k 个节点不能翻转”的处理。更容易理解的模式是：先确认下一组长度足够，再把这一段从原链表中临时断开，调用普通的 reverseList 翻转，最后把上一段、翻转段、下一段重新接起来。

官方题解对比

LeetCode 官方题解提供了模拟式的链表分组翻转思路：用哑节点维护答案头部，逐组定位 head 与 tail，翻转当前闭区间后接回原链表。这里采用同一复杂度级别的迭代写法，时间复杂度 O(n)，额外空间 O(1)；它把“翻转一组”和“接回链表”拆开讲，比把翻转边界揉进同一个循环里更直观。

解题思路

暴力地把链表节点放进数组再按组改顺序很直观，但需要额外空间，而且没有练到链表指针操作。优化做法是只在链表上移动引用。

使用 dummy 指向原头节点，prevGroupEnd 表示上一组翻转后的尾节点，也就是当前待处理组的前驱。每轮先从 prevGroupEnd 出发向后走 k 步找到 kthNode。如果过程中遇到 null，说明剩余节点不足 k 个，直接返回答案。

当一组完整存在时，先保存 groupStart = prevGroupEnd.next 和 nextGroupStart = kthNode.next。然后执行 kthNode.next = null，把当前组临时变成一条独立链表，调用普通的 reverseList(groupStart)。翻转后，prevGroupEnd.next 接新组头，groupStart.next 接下一组开头，最后把 prevGroupEnd 移到 groupStart 继续下一轮。

参考实现

package com.leetcode2026.reverse_nodes_in_k_group_025;

public class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        // 空链表或 k = 1 时不需要改动；提前返回也避免后续循环做无意义工作。
        if (head == null || k <= 1) {
            return head;
        }

        // dummy 让第一组翻转后也能通过“上一组尾节点”统一接回。
        ListNode dummy = new ListNode(0, head);
        ListNode prevGroupEnd = dummy;

        while (true) {
            // 从上一组尾节点向后数 k 个，确认当前组是否完整。
            ListNode kthNode = prevGroupEnd;
            for (int i = 0; i < k; i++) {
                kthNode = kthNode.next;
                if (kthNode == null) {
                    return dummy.next;
                }
            }

            // groupStart 会在翻转后变成当前组的尾；nextGroupStart 用来接回后半段。
            ListNode groupStart = prevGroupEnd.next;
            ListNode nextGroupStart = kthNode.next;

            // 断开当前组，让 reverseList 可以按普通整链表反转来处理。
            kthNode.next = null;
            prevGroupEnd.next = reverseList(groupStart);
            groupStart.next = nextGroupStart;

            // 旧组头已经成为新组尾，下一轮从这里继续找 k 个节点。
            prevGroupEnd = groupStart;
        }
    }

    private ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }
        return prev;
    }
}

核心变量/状态

变量含义更新时机
dummy固定的哑节点，避免第一组翻转后丢失新头。初始化后不再移动。
prevGroupEnd上一组翻转后的尾节点，也是当前组的前驱。每组翻转后移动到 groupStart。
kthNode当前组第 k 个节点，用来判断这一组是否完整。每轮从 prevGroupEnd 向后走 k 步。
groupStart当前组原本的头节点，翻转后会变成这一组的尾节点。翻转前保存，翻转后用它接 nextGroupStart。
nextGroupStart下一组的头节点。执行 kthNode.next = null 前必须先保存。
reverseList普通的整链表反转函数。当前组被临时断开后调用。

执行过程可视化

易错点

  • 没有先检查剩余节点是否达到 k 个，导致尾部不足一组也被翻转。
  • 执行 kthNode.next = null 前没有保存 nextGroupStart，导致翻转后找不到下一组入口。
  • 翻转后忘记让 groupStart.next 接回 nextGroupStart，链表会在当前组尾部断掉。
  • 把 prevGroupEnd 移到新组头，实际上下一轮前驱应该是旧组头 groupStart，也就是当前组翻转后的尾节点。
  • k = 1 时应该保持原链表，同时循环仍要能正常推进。
  • 测试只看节点值，不检查是否产生环，容易漏掉指针接错造成的死循环。

复杂度分析

每个节点在查找分组和组内翻转中被访问常数次，因此时间复杂度是 O(n)。只使用若干引用变量，额外空间复杂度是 O(1)。
