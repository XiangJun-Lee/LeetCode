LeetCode 15：三数之和

排序后固定第一个数，把剩余问题变成有序数组上的双指针查找，并在三个层面跳过重复值。

中等 数组 双指针 排序 时间 O(n^2)

题目概述

给定整数数组 nums，返回所有不重复的三元组 [nums[i], nums[j], nums[k]]，要求三个下标互不相同，并且三个数之和等于 0。答案中的三元组不能重复，输出顺序不重要。

题号：15
标题：三数之和
约束：3 <= nums.length <= 3000

示例拆解

示例 1

nums = [-1,0,1,2,-1,-4]

排序后是 [-4,-1,-1,0,1,2]。固定 -1 时，左右指针可以找到 [-1,-1,2] 和 [-1,0,1]。

示例 2 与 3

[0,1,1] 没有任意三数和为 0，返回空列表。[0,0,0] 只有一个不重复三元组 [0,0,0]。

考察点

  • 排序：让相同元素相邻，也让双指针移动方向有单调依据。
  • 双指针：固定 i 后，用 left 和 right 在剩余区间里找两数和。
  • 去重：固定值、左指针值、右指针值都要跳过重复，否则会产生重复三元组。

官方题解对比

官方题解的核心路线是先排序，再枚举第一个数，最后在右侧区间用双指针寻找互补的两个数。这里采用相同的最优主思路，时间复杂度保持 O(n^2)，额外空间除答案外为 O(log n) 到 O(n)，取决于排序实现。

暴力三重循环是 O(n^3)，在 n = 3000 时不可接受。排序以后，第二、第三个数不需要重新嵌套枚举：和太小就移动左指针增大和，和太大就移动右指针减小和。

解题思路

排序
枚举 i
跳过重复 i
双指针收缩
命中后去重
  1. 先对 nums 升序排序。
  2. 枚举 i 作为三元组的第一个位置。如果 nums[i] > 0，后面都不可能凑成 0，可以提前结束。
  3. 若 i > 0 且 nums[i] == nums[i - 1]，跳过，避免固定相同的第一个数。
  4. 设置 left = i + 1、right = nums.length - 1。若三数和小于 0，左指针右移；若大于 0，右指针左移。
  5. 命中 0 后收集结果，并跳过左右两侧相同的值，再继续向内收缩。

参考实现

package com.leetcode2026.three_sum_015;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

public class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> answer = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            // 排序后，如果第一个数已经大于 0，后面只会更大，不可能得到和为 0 的三元组。
            if (nums[i] > 0) {
                break;
            }

            // 固定相同的第一个数会得到重复答案，必须从源头跳过。
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int left = i + 1;
            int right = nums.length - 1;

            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    answer.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // 命中后，左右指针都要跳过相同值，避免同一个三元组重复出现。
                    int leftValue = nums[left];
                    int rightValue = nums[right];
                    while (left < right && nums[left] == leftValue) {
                        left++;
                    }
                    while (left < right && nums[right] == rightValue) {
                        right--;
                    }
                } else if (sum < 0) {
                    // 和偏小，需要更大的数，所以移动左指针。
                    left++;
                } else {
                    // 和偏大，需要更小的数，所以移动右指针。
                    right--;
                }
            }
        }

        return answer;
    }
}

核心变量/状态

变量含义更新时机
i固定的第一个数位置外层循环推进，遇到重复值直接跳过
left第二个数候选位置当前和小于 0 或命中后去重时右移
right第三个数候选位置当前和大于 0 或命中后去重时左移
answer已经找到的不重复三元组仅在三数和等于 0 时追加

执行过程可视化

示例数组排序后：[-4,-1,-1,0,1,2]。点击下一步观察固定值和左右指针如何移动。

易错点

  • 只对答案做集合去重，而不是在指针移动时去重，会让逻辑更复杂，也容易漏掉顺序归一化。
  • 命中一个三元组后只移动一侧指针，会在重复值密集时产生重复答案或停留过久。
  • 忘记跳过重复的 i，例如两个 -1 会各自生成 [-1,0,1]。
  • 使用 left <= right 会允许同一个元素被使用两次，必须保持 left < right。

复杂度分析

排序需要 O(n log n)。外层枚举 i，内层双指针每轮线性收缩，因此主复杂度是 O(n^2)。除返回答案外，额外空间主要来自排序过程，Java 的基本类型数组排序通常为 O(log n) 级递归栈；若按通用排序实现口径，也可写作 O(log n) 到 O(n)。
