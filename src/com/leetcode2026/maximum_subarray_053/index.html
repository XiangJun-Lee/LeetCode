LeetCode 53：最大子数组和

在整数数组中找到和最大的非空连续子数组，只返回这个最大和。

数组 动态规划 Kadane 算法 分治

题目概述

给定整数数组 nums，请找出一个具有最大和的连续子数组，子数组最少包含一个元素，返回其最大和。连续是关键：可以截取一段相邻元素，但不能跳着选。

本题源自 LeetCode 53 最大子数组和，官方题解提供了动态规划和分治两种主线。练习时优先掌握 O(n) 时间、O(1) 空间的一维动态规划写法。

示例拆解

示例 1

nums = [-2,1,-3,4,-1,2,1,-5,4]

输出 6，因为连续子数组 [4,-1,2,1] 的和最大。

示例 2

nums = [1]

输出 1。非空数组只有一个元素时，答案就是它本身。

示例 3

nums = [5,4,-1,7,8]

输出 23。整段数组虽然包含一个负数，但整体收益仍最大。

负数边界

nums = [-2,-1,-3]

输出 -1。不能返回空数组的 0，必须选择一个非空子数组。

考察点

  • 数组连续区间：答案必须来自一段相邻元素。
  • 动态规划：维护“以当前位置结尾的最大子数组和”。
  • Kadane 算法：当旧前缀已经拖累当前值时，从当前元素重新开始。
  • 分治：官方也给出线段信息合并思路，可扩展到区间查询类问题。
  • 边界处理：全负数、单元素、包含 0、最优段位于开头或末尾。

官方题解对比

方法 核心状态 复杂度 练习建议
动态规划 / Kadane current = max(nums[i], current + nums[i])，best = max(best, current) 时间 O(n)，空间 O(1) 作为主解法，代码短但非常考验初始化和更新顺序。
分治 每段维护区间和、最大前缀和、最大后缀和、最大子段和。 时间 O(n)，递归空间 O(log n) 适合理解线段树合并思想，不是本练习文件的首选实现。
暴力枚举 枚举左右端点并求和。 通常 O(n²) 或 O(n³) 只用于推导，不满足官方最优复杂度。

解题思路

  1. 如果枚举每个子数组，重复计算太多；我们真正需要的是“到当前位置为止，必须以当前位置结尾的最好结果”。
  2. 设 current 表示以当前元素结尾的最大连续和。来到 nums[i] 时只有两种选择：接在前一段后面，或者从当前元素重新开始。
  3. 因此 current = Math.max(nums[i], current + nums[i])。如果前一段为负，重新开始更优；如果前一段为正，继续累加更优。
  4. best 记录扫描过程中见过的最大 current，最终返回 best。

初始化要用 nums[0]，不能用 0，否则全负数数组会错误返回 0。

参考实现

package com.leetcode2026.maximum_subarray_053;

public class Solution {
    public int maxSubArray(int[] nums) {
        // current 表示“必须以当前下标结尾”的最大连续子数组和。
        int current = nums[0];
        // best 表示扫描到当前位置时，全局见过的最大连续子数组和。
        int best = nums[0];

        for (int i = 1; i < nums.length; i++) {
            int value = nums[i];
            // 如果 current + value 不如 value 自己，说明前面的连续段已经变成负担。
            current = Math.max(value, current + value);
            // current 更新后才有资格参与全局答案，因为它代表以 i 结尾的最优段。
            best = Math.max(best, current);
        }

        return best;
    }
}

核心变量/状态

current

必须以当前位置结尾的最大子数组和。它不是全局答案，而是下一轮状态转移的依据。

best

扫描过程中出现过的最大 current。每轮更新 current 后再更新它。

nums[i]

当前元素。它决定是重启连续段，还是接到上一段后面。

易错点

  • 把答案初始化为 0，导致全负数用例错误。
  • 误以为负数一定要断开；如果前面累计值仍为正，跨过小负数可能得到更大和。
  • 把“子数组”和“子序列”混淆，错误跳过中间元素。
  • 先更新 best 再更新 current，会漏掉以当前元素结尾的新答案。

复杂度分析

时间复杂度：O(n)

只线性扫描数组一次。

空间复杂度：O(1)

只维护两个整数状态，不需要额外数组。
