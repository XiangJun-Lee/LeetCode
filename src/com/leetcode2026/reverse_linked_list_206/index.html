LeetCode 206：反转链表

给定单链表头节点，把链表原地反转并返回新的头节点。核心是保存后继节点，再把当前节点的 next 指向前驱。

Java 链表 双指针 迭代

题目概述

输入一个单链表的头节点 head，返回反转后的链表头节点。链表节点定义为 ListNode，每个节点只有 val 和 next。

反转不是交换节点值，而是改变每个节点的 next 指针方向。

示例拆解

输入链表 输出链表 解释
1 → 2 → 3 → 4 → 5 5 → 4 → 3 → 2 → 1 所有节点方向反过来，原尾节点 5 变成新头节点。
1 → 2 2 → 1 2 的 next 指向 1，1 的 next 指向 null。
[] [] 空链表没有节点，直接返回 null。

考察点

链表指针操作

理解节点之间不是数组下标关系，而是由 next 串起来。

双指针迭代

prev 指向已反转部分头部，curr 指向当前待处理节点。

原地修改

不需要新建整条链表，只改变每个节点的指向。

官方题解对比

官方题解主要给出迭代和递归两种做法。这个页面以迭代法为主，因为它空间复杂度为 O(1)，更适合作为面试首选答案；递归写法更短，但调用栈会占用 O(n) 空间。

本页方案

使用 prev、curr、next 三个引用，循环中逐个反转节点指向。

复杂度不弱于官方最优

时间复杂度 O(n)，额外空间 O(1)，和官方迭代方案一致。

解题思路

保存 curr.next
curr.next 指向 prev
prev 前进到 curr
curr 前进到 next

循环过程中，prev 始终代表“已经反转好的链表头”，curr 代表“还没反转的第一个节点”。当 curr 走到 null，说明所有节点都处理完，prev 就是新的头节点。

参考实现

class Solution {
    public ListNode reverseList(ListNode head) {
        // prev 指向已经反转完成的链表头；一开始还没有节点，所以是 null。
        ListNode prev = null;

        // curr 指向当前要处理的节点，从原链表头开始。
        ListNode curr = head;

        while (curr != null) {
            // 先保存原来的后继节点，否则 curr.next 改掉后会丢失剩余链表。
            ListNode next = curr.next;

            // 反转当前节点指向：让 curr 接到已经反转好的链表前面。
            curr.next = prev;

            // prev 前进，新的已反转链表头变成 curr。
            prev = curr;

            // curr 前进，继续处理原链表剩余部分。
            curr = next;
        }

        // curr 为 null 时，prev 指向新链表头。
        return prev;
    }
}

核心变量/状态

变量 含义 关键不变量
prev 已反转部分的头节点 每处理完一个节点，prev 都会成为新的反转链表头
curr 当前待反转节点 只要 curr 不为空，循环就继续
next curr 原本的后继节点 必须在改 curr.next 前保存，否则剩余链表会丢

易错点

  1. 先改 curr.next，后保存 next：会丢失原链表剩余部分。
  2. 返回 curr：循环结束时 curr 是 null，应该返回 prev。
  3. 忘记处理空链表：迭代法天然支持空链表，head 为 null 时直接返回 null。
  4. 只交换 val：题目要求反转链表结构，不是交换节点值。
  5. 递归忘记断开 head.next：递归法里不处理好可能形成环。

复杂度分析

时间复杂度：O(n)

每个节点只访问一次，循环处理 n 个节点。

空间复杂度：O(1)

迭代法只使用固定数量的引用变量，不随链表长度增长。
