LeetCode 215：数组中的第K个最大元素

把“第 k 大”转换成升序数组中的下标 n - k，用快速选择只定位目标，不完整排序。

数组 快速选择 分治 堆 排序

题目概述

给定整数数组 nums 和整数 k，返回数组中第 k 个最大的元素。

注意这里找的是排序后的第 k 个最大值，不是第 k 个不同的值。

关键转换：第 k 大等价于升序排序后下标 nums.length - k 的元素。

例如长度为 6，找第 2 大，就找升序下标 4。

示例拆解

示例一

nums = [3,2,1,5,6,4], k = 2

升序结果是 [1,2,3,4,5,6]，第 2 大是 5。

示例二

nums = [3,2,3,1,2,4,5,5,6], k = 4

升序结果是 [1,2,2,3,3,4,5,5,6]，第 4 大是 4。

考察点

快速选择

只关心目标下标落在哪边，每次 partition 后丢掉另一半。

分治

用 pivot 把数组分成小于区和大于区，递归或循环缩小搜索区间。

堆

也可维护大小为 k 的小顶堆，堆顶就是当前第 k 大。

官方题解对比

方法 复杂度 适用场景
完整排序 O(n log n) 最容易写，但没有利用“只找一个位置”的特点。
小顶堆 O(n log k) 适合数据流或不想修改原数组的场景。
随机化快速选择 平均 O(n) 本页参考实现。只把目标元素放到正确位置，不保证两侧有序。

解题思路

第 k 大
转成 target = n - k
partition 定 pivot
缩小左右边界

快速选择和快速排序共用 partition，但目标不同：快速排序要让两边继续有序；快速选择只需要判断 pivot 的最终位置和 target 的关系。

  • 如果 pivotIndex == target，答案就是 nums[pivotIndex]。
  • 如果 pivotIndex < target，目标在右侧，令 left = pivotIndex + 1。
  • 如果 pivotIndex > target，目标在左侧，令 right = pivotIndex - 1。

参考实现

import java.util.Random;

class Solution {
    private final Random random = new Random();

    public int findKthLargest(int[] nums, int k) {
        // 第 k 大转换为升序数组中的目标下标。
        // 例如 n = 6, k = 2，目标是下标 4，也就是第二大的元素。
        int target = nums.length - k;
        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            // 随机 pivot 能降低退化到 O(n^2) 的概率。
            int pivotIndex = left + random.nextInt(right - left + 1);
            int finalIndex = partition(nums, left, right, pivotIndex);

            // partition 后，finalIndex 左边都小于 pivot，
            // 右边都大于等于 pivot，因此 pivot 已经在升序中的正确位置。
            if (finalIndex == target) {
                return nums[finalIndex];
            }
            if (finalIndex < target) {
                // pivot 太靠左，说明第 k 大还在右半边。
                left = finalIndex + 1;
            } else {
                // pivot 太靠右，说明目标在左半边。
                right = finalIndex - 1;
            }
        }

        return -1;
    }

    private int partition(int[] nums, int left, int right, int pivotIndex) {
        int pivot = nums[pivotIndex];

        // 先把 pivot 放到末尾，方便扫描区间 [left, right - 1]。
        swap(nums, pivotIndex, right);

        // storeIndex 表示下一个“小于 pivot”的元素应该放置的位置。
        int storeIndex = left;
        for (int i = left; i < right; i++) {
            if (nums[i] < pivot) {
                swap(nums, storeIndex, i);
                storeIndex++;
            }
        }

        // 把 pivot 放回分界点，此时它左边都小于它，右边都大于等于它。
        swap(nums, storeIndex, right);
        return storeIndex;
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

核心变量/状态

变量 含义 更新时机
target 第 k 大在升序数组中的下标 初始化为 nums.length - k，之后不变
left/right 当前还可能包含答案的闭区间 根据 finalIndex 和 target 的大小关系收缩
pivot 本轮分区基准值 每轮随机选择，partition 后落到最终位置
storeIndex 小于 pivot 区域的下一个写入位置 每遇到一个小于 pivot 的元素就右移一格

易错点

  • 把第 k 大当成下标 k：升序目标下标是 n - k。
  • partition 后继续查两边：快速选择只查目标所在的一边。
  • 忘记重复元素：题目不是找第 k 个不同元素，重复值正常参与排名。
  • 固定 pivot：有序或接近有序数组容易退化，随机 pivot 更稳。
  • 误以为两边有序：partition 只保证大小关系，不保证左右子数组内部有序。

复杂度分析

时间复杂度

随机化快速选择平均 O(n)，最坏 O(n^2)。

空间复杂度

迭代写法只使用常数额外变量，空间复杂度 O(1)。
